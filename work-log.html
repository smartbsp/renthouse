# 工作日誌 2026-06-05 — V2.1 水費+搜尋+修復 ## 版本 - V1.1: 穩定基礎版 - V1.11: +水費計算器 +即時財務 +手機滑動 - V2.1: +房客搜尋(修復定位) --- ## 關鍵教訓 ### 頁籤結構定位 - **問題**: 多次因 `` 定位失誤導致結構崩塌，搜尋卡誤入電費頁籤 - **解法**: 使用獨特標記(id)定位，不依賴模糊的 `` 匹配 - **正確定位**: `editBtnB` 唯一標記 → 搜尋卡插在 card close 與 tab-tenant close 之間 --- ## 死當/崩潰原因分析 ### 1. 多重 `` 匹配失誤（最嚴重） - **現象**: 插入 HTML 時用 `\n\n` 當 oldString，檔案中有多處相同組合 - **結果**: 編輯誤配到錯誤位置，刪除或移動了關鍵的 ``，破壞 DOM 結構 - **症狀**: 頁面空白、頁籤內容消失、功能出現於錯誤頁籤 - **修復**: 改用獨特 id（如 `editBtnB`）當錨點，不再用泛泛的 `` 匹配 ### 2. 未經測試一次大量修改 - **現象**: 同時改 HTML + CSS + JS，出錯無法定位哪一項導致 - **結果**: 反覆 git revert 浪費時間 - **修復**: 改為一次只改一個功能，上傳測試後再改下一個 ### 3. SW 快取混淆問題診斷 - **現象**: 檔案已正確上傳但瀏覽器顯示舊版 - **誤判**: 多次以為是程式碼錯誤，實際是 Service Worker 快取 - **辨別方法**: 換瀏覽器測試、curl 查看原始回應、比對 MD5 ### 4. modalTitle 缺 id - **現象**: `showSearchDetail` 存取 `getElementById('modalTitle')` 取到 null - **影響**: 點擊搜尋結果時才觸發，非頁面載入時，難以排查 - **修復**: 在 modal h2 加 `id="modalTitle"` ### 5. 箭頭函式與舊瀏覽器相容 - **現象**: 程式碼中有 `=>` 箭頭函式，部分環境不支援 - **修復**: 改用 `function(){}` 寫法，避免未知崩潰 ### 水費頁籤 - 完整水費帳單格式: 抄表日、指針數、公共分攤度數、基本費/用水費/代徵費用 - 本期實用度數 = 本期指針數 − 上期指針數 + 公共分攤度數 - 計算模式: `依實際住戶人數分攤` / `依實際用水度數分攤` - 模式選擇器在右上角，按鈕(新增/編輯)在右下角 - 編輯/儲存共用按鈕切換，小尺寸(`class="small"`)靠右 ### 水費歷史 - 仿電費: 月份/期間過濾、累計摘要、點選編輯、刪除 - API: `water_save`, `water_list`, `water_delete` - MySQL: `renthouse_water_records` 表 ### 即時計算 - 財務欄位全部改用 `oninput`(非 `onchange`) - `updateFinTotal` 結尾呼叫 `calcPayDates` → 每期應繳/實收同步更新 ### 手機滑動 - CSS `scroll-snap` + swiper 容器 - `switchTab` 配合 `scrollTo` - scroll 事件監聽同步頂部頁籤按鈕 --- ## 資料庫 | 表 | 用途 | |----|------| | renthouse_records | 電費 | | renthouse_tenants | 房客(JSON) | | renthouse_tenant_log | 房客異動 | | renthouse_water_records | 水費 | | renthouse_drafts | 電費草稿 | ## 部署 - `scp index.html api.php sw.js nas:/share/CACHEDEV1_DATA/Web/renthouse/` - `上傳NAS.bat` - 備份: `git tag v*.*` + `ssh nas cp` + `mysqldump`
